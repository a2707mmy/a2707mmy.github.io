    二、常用變數:

  6. int rows:像素陣列的列數。
  7. int cols:像素陣列的行數。
  8. uchar* data:uchar類型的指標，指向像素陣列的首個地址。
  9. int dims:陣列的維度，以圖像來說為2。

三、 常用函數:

  1. int channels( ):回傳通道數，1為灰度圖， 3為BGR圖， 4為BGRA圖(A為透明度)。
  2. int depth( ):回傳一整數，用以表示像素的精度。


3. int type( ):回傳一整數，用以表示像素的資料型態。


4. imread(const char path,int flag):會以flag所對應的讀取方式讀入path路徑的檔案，詳細見下面”從圖檔讀取內容”。
5. step1():會回傳一列像素總共有幾筆資料，相當於cols*channel。
6. Iscontinue():回傳像素是否連續，這是因為圖像資料每一行中有可能有空，或是0，例如以12位元存資料，但資料內容只有8位元，那就會有4位元為0。可以Iscontinue來檢查資料是否連續。

四、 像素的資料型態:

Mat宣告時，可以透過type來指定像素陣列的資料型態。
命名方式為CV_(bit)(data type)C(channel)，bit代表矩陣中一個數值的位元數，data type代表矩陣中的數值類型，channel代表通道數。


注意!這裡的type是指宣告時的其中一個引數，並不是函數介紹中傳回的數值。Mat物件實際上也沒有type變數可以供使用者調用，在宣告完成之後要改變像素資料型態，必須要透過covert的方式，暫不討論。

2.讀取影像-imread

使用者可以透過imread函數，從圖像檔案(BMP、JPG……)讀取資料進入Mat物件。
需要注意的是imread會以引數flag來決定讀取方式，並對Mat物件的各種變數做改變，也就是如果要確保存於Mat物件的資料結構無誤，應該要注意imread中flag的使用，而非宣告Mat的方式。

  1. 第一個引數是檔案名稱。
  2. 第二個引數flags是讀取的方式。根據讀取的方式不同，會在讀入影像時對影像做不同的處理，通常是轉換成灰階或是彩色。
    例如:
    IMREAD_UNCHANGED:直接讀入不對影像處理，讀取圖片中所有的 channels包含透明度。若Mat物件沒有透明度通道，透明度數值會被捨棄。
    IMREAD_GRAYSCALE:讀入時轉換成灰階資料。
    IMREAD_COLOR:以BGR的資料型態讀入，此為預設值。
  3. 目前確定JPG、BMP以IMREAD_UNCHANGED讀入時，資料型態為CV_8UC3:每個像素由3個無符號的整數構成(0~255)，分別對應BGR。
    注意!OpenCV Mat彩色圖像，其彩色數據內容排列為BGR，而非RGB。

可以透過副檔名指定檔案格式，並透過引數params指定輸出時的品質。

    For JPEG, it can be a quality ( CV_IMWRITE_JPEG_QUALITY ) from 0 to 100 (the higher is the better). Default value is 95.
    For PNG, it can be the compression level ( CV_IMWRITE_PNG_COMPRESSION ) from 0 to 9. A higher value means a smaller size and longer compression time. Default value is 3.
    For PPM, PGM, or PBM, it can be a binary format flag ( CV_IMWRITE_PXM_BINARY ), 0 or 1. Default value is 1.
